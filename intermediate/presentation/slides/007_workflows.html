Workflows

TFE supports different workflows to manage infrastructure changes and deployments:

Workflow Description Benefits Drawbacks
Version Control Workflow Integrates with version control systems (e.g., Git) to trigger runs based on code changes.
  • Automated
  • Auditable
  • Supports collaboration
  • CI/CD best practices
  • Requires VCS integration setup
  • Less flexibility for ad-hoc changes
CLI Workflow Allows users to manually trigger runs and apply changes via cli.
  • Simple to use
  • Flexible for quick or one-off changes
  • Prone to human error
  • Versed working with a CLI
  • Requires CI/CD integration
API Workflow Enables programmatic control over runs and deployments through the TFE API.
  • Highly flexible
  • Enables automation
  • Integration with other systems
  • Requires API knowledge
  • Custom tooling needed
  • Can be complex to manage securely
